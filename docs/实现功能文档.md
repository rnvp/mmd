# MMD 实现功能文档

本文档定义当前项目第一阶段要实现的产品形态、界面风格、数据结构与开发边界，目标是在实现前先把范围收紧，避免一边写一边漂移。

## 1. 项目定位

`MMD` 是一个基于 `Tauri + React` 的桌面 Markdown 编辑器。

第一阶段目标不是做知识库，也不是做全功能 IDE，而是先完成一个：

- 默认双栏
- 左侧编辑、右侧预览
- 本地文件可打开、编辑、保存
- 具备基础编辑体验
- 风格参考 `tiledial`

的桌面 Markdown 编辑器。

## 2. 参考样式基线

本项目界面风格参照上层目录中的 `tiledial`，但不是直接复制业务结构。需要继承的是它的视觉语言：

- 深色桌面工具风格
- 无边框自定义标题栏
- 低饱和背景与细边框分层
- 卡片化面板
- 紧凑但不拥挤的工具栏
- 悬浮高亮而非强烈发光

需要抽象成适合编辑器的界面规则：

- 整体使用深灰到近黑背景，不做纯黑。
- 编辑区与预览区分成两个主面板，中间有明显但克制的分隔。
- 工具栏高度控制在单行，优先轻量按钮，不堆叠复杂控制项。
- 面板圆角、描边、hover 高亮强度参考 `tiledial` 的卡片交互。
- 排版优先可读性，正文与代码字体分离。

## 3. 第一阶段功能范围

第一阶段只做“可稳定使用的基础 Markdown 编辑器”，包含以下内容。

### 3.1 默认双栏布局

- 桌面端默认双栏。
- 左栏为 Markdown 源码编辑区。
- 右栏为实时预览区。
- 中间支持拖拽调整宽度。
- 最小宽度需要受限，避免某一侧被压到不可用。

### 3.2 基础文件能力

- 新建空文档
- 打开本地 Markdown 文件
- 保存当前文件
- 另存为
- 显示当前文件路径或文件名
- 对“未保存修改”提供状态提示
- 默认使用 `UTF-8` 编码读写文本文件
- 在新建、打开其他文件、关闭窗口前拦截未保存修改
- 支持将 `.md` / `.markdown` / `.txt` 文件拖入窗口后直接打开

第一阶段默认支持扩展名：

- `.md`
- `.markdown`
- `.txt`

### 3.3 基础编辑能力

- 文本输入
- 撤销 / 重做
- Tab 缩进
- Shift+Tab 反向缩进
- 常用快捷键支持
- 当前文档脏状态跟踪
- 自动滚动保持编辑体验稳定
- 对 `Codex` 使用场景有良好支持

这里的“良好支持 Codex”至少包括：

- 默认按 `UTF-8` 打开和保存文档
- 能稳定编辑 `Markdown`、提示词、代码片段和命令文本
- 保持纯文本可控，不引入破坏格式的富文本行为
- 对较长上下文文本保持稳定输入和预览表现

当前第一版已接入这些快捷键：

- `Ctrl/Cmd+B` 粗体
- `Ctrl/Cmd+I` 斜体
- `Ctrl/Cmd+K` 链接
- `Ctrl/Cmd+E` 行内代码
- `Ctrl/Cmd+1` 标题

### 3.4 Markdown 实时预览

- 编辑后预览区实时刷新
- 编辑区和预览区支持基础同步滚动
- 支持常见 Markdown 语法：
  - 标题
  - 段落
  - 粗体 / 斜体
  - 引用
  - 有序 / 无序列表
  - 行内代码
  - 代码块
  - 分隔线
  - 链接
  - 图片
  - 表格

### 3.5 视图切换

虽然默认是双栏，但仍提供基础模式切换：

- 双栏
- 仅编辑
- 仅预览
- 当前视图模式和分栏比例会在本地持久化
- 当前主题模式会在本地持久化

移动或极窄窗口下可自动退化为单栏，但桌面默认打开必须是双栏。

### 3.6 基础插入与格式化动作

工具栏至少提供这些快捷动作：

- 标题
- 粗体
- 斜体
- 行内代码
- 代码块
- 链接
- 图片
- 无序列表
- 有序列表
- 引用

这些动作本质上是对选区做文本包裹或插入模板，不引入复杂富文本编辑。

当前第一版实现里，常用格式化按钮直接放在标题栏，更多操作收口到 `More` 菜单。

### 3.7 主题切换

- 提供 `dark / light` 两套主题
- 标题栏可直接切换主题
- 当前主题在本地持久化
- Markdown 预览、状态栏、弹窗与拖拽态需要跟随主题统一切换

## 4. 第一阶段不做的内容

以下内容明确不进入当前阶段：

- 云同步
- 账号系统
- 前端硬编码任何 API 密钥
- 多标签页
- 工作区树状目录
- 全文搜索
- 插件系统
- Vim 模式
- 所见即所得富文本编辑
- 导出 PDF / HTML
- 协同编辑
- AI 生成功能

## 5. 推荐技术实现

在不改变总体技术栈的前提下，第一阶段建议如下：

- 前端：`React + TypeScript`
- 构建：`Vite`
- 桌面壳：`Tauri`
- 样式：`Tailwind CSS v4`
- 组件：`shadcn/ui`
- Markdown 解析渲染：优先选稳定、维护良好的成熟库
- 文件打开/保存：优先通过 Tauri 能力完成

说明：

- 用户要求尽量使用最新版包，因此实现时优先选当前稳定大版本。
- 如果只是做基础编辑，不必一开始引入过重的编辑器生态；应先选足够稳定、可控、集成成本低的方案。
- 共用逻辑需要拆分，例如文档状态、文件 IO、Markdown 渲染配置、工具栏动作，不要全部堆进一个组件。

## 6. 前端结构建议

第一阶段建议至少拆成以下层次：

- `AppShell`
  - 负责窗口级布局、标题栏、状态栏
- `EditorPane`
  - 负责源码输入
- `PreviewPane`
  - 负责 Markdown 预览渲染
- `Toolbar`
  - 负责插入和视图切换
- `DocumentStore` 或等价状态层
  - 负责文件路径、内容、脏状态、保存逻辑

共用逻辑建议拆分为：

- 文档默认值与状态标准化
- 文件读写 API
- Markdown 渲染配置
- 编辑器文本操作工具函数
- 视图模式管理

## 7. 窗口与布局规范

界面应参考 `tiledial` 的桌面感，但改成编辑器布局。

### 7.1 标题栏

- 使用自定义标题栏
- 左侧展示应用名与当前文件名
- 中部保留可拖动区域
- 功能按钮和菜单统一放在虚拟标题栏中
- 不单独增加传统顶层菜单栏
- 当前标题栏需要承载核心动作入口，例如：
  - 新建
  - 打开
  - 保存
  - 另存为
  - 视图切换
  - 常用 Markdown 格式化动作
  - 更多菜单
  - 窗口控制按钮
- 若使用 Tauri v2 的前端窗口 API，需显式放行对应 `window` 权限，否则拖动与窗口控制按钮可能无效

### 7.2 主体区

- 主体区不再额外放传统菜单栏
- 中间为编辑区 / 预览区
- 底部可选状态栏

当前第一版实现中，格式化动作也已经收口到虚拟标题栏，不再单独提供第二条工具栏。

### 7.3 视觉规范

- 编辑区和预览区都需要独立背景层
- 代码块、按钮、输入框统一边框和圆角语言
- hover、focus、active 使用统一语义色
- 禁止出现默认浏览器原生样式拼贴感
- 深浅主题都要保持同一套层级与语义色映射，不允许只切背景不修正文案与边框对比度

## 8. 数据模型建议

第一阶段文档状态建议至少包含：

```ts
type ViewMode = 'split' | 'editor' | 'preview';
type ThemeMode = 'light' | 'dark';

type DocumentState = {
  filePath: string | null;
  fileName: string;
  content: string;
  savedContent: string;
  isDirty: boolean;
  viewMode: ViewMode;
  editorRatio: number;
};
```

约束：

- `savedContent` 用于判断是否有未保存修改。
- `fileName` 在无路径时可回退为 `Untitled.md`。
- `editorRatio` 代表双栏分割比例，允许持久化。
- `ThemeMode` 允许独立持久化，不强塞进文档内容模型。
- 文件读写默认按 `UTF-8` 处理。

## 9. 基础交互要求

### 9.1 新建

- 新建后清空当前内容
- 若当前文档未保存，需要先处理脏状态

### 9.2 打开

- 通过系统文件选择器打开文本文件
- 打开后更新编辑区、预览区、标题栏

### 9.3 保存

- 已有关联路径时直接保存
- 无路径时走另存为

### 9.4 关闭前保护

- 若存在未保存修改，需要提示用户
- 至少在“新建”、“打开其他文件”、“关闭窗口”前生效
- 当前实现使用应用内三按钮对话框：
  - Save and Continue
  - Discard
  - Cancel

## 10. 文档与工程维护要求

按当前仓库约束，后续实现时需要同步维护：

- `README.md`
- `docs/实现功能文档.md`
- `docs/开发者复刻-checklist.md`

文档更新原则：

- 先定义范围，再写代码
- 功能范围变化时同步更新文档
- 实际实现如果偏离本文档，需要立刻修正文档，而不是放任失真

## 11. 一句话总结

第一阶段的 `MMD` 要做成一个带 `tiledial` 风格气质的桌面 Markdown 编辑器：默认双栏、文件可用、预览实时、操作克制、结构清晰，先把基础编辑体验做稳。
