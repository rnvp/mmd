# MMD 开发者复刻 Checklist

这份清单只服务于第一阶段实现，不解释原理。原理和范围以 [实现功能文档.md](/D:/now/codex/mmd/docs/实现功能文档.md) 为准。

## 1. 基础工程

- 建立 `Tauri + React + TypeScript + Vite` 项目结构。
- 接入 `Tailwind CSS v4`。
- 接入 `shadcn/ui`。
- 所有源码文件使用 `UTF-8`。
- 不在前端硬编码任何 API 密钥或敏感凭证。

## 2. 应用骨架

- 创建自定义标题栏。
- 功能按钮和菜单统一放入自定义标题栏。
- 不单独创建传统菜单栏。
- 创建主编辑区。
- 创建预览区。
- 创建可选状态栏。
- 默认进入双栏模式。

## 3. 风格对齐

- 视觉风格参考 `tiledial`。
- 使用深色桌面工具风格。
- 使用低饱和背景、细边框、圆角卡片。
- 按钮和输入框保持统一语义色与交互状态。
- 避免浏览器默认表单样式直接裸露。

## 4. 文档状态

- 维护当前文件路径。
- 维护当前文件名。
- 维护当前 Markdown 内容。
- 维护上次保存内容。
- 维护 `isDirty`。
- 维护视图模式：`split | editor | preview`。
- 维护双栏分割比例。
- 维护主题模式：`light | dark`。

## 5. 文件能力

- 新建文档。
- 打开本地文件。
- 保存当前文档。
- 另存为。
- 支持 `.md` / `.markdown` / `.txt`。
- 默认按 `UTF-8` 读写文本文件。
- 标题栏显示当前文件名。
- 新建 / 打开 / 关闭前处理未保存修改。
- 支持拖拽 Markdown / 文本文件打开。

## 6. 未保存保护

- 新建前检查脏状态。
- 打开其他文件前检查脏状态。
- 窗口关闭前检查脏状态。
- 提供保存 / 不保存 / 取消的处理路径。

## 7. 编辑器基础能力

- 支持基础文本输入。
- 支持撤销 / 重做。
- 支持 Tab 缩进。
- 支持 Shift+Tab 反向缩进。
- 支持常用快捷键。
- 支持选区插入模板。
- 对 `Codex` 编写场景保持良好支持。
- 不引入会破坏纯文本结构的富文本编辑行为。

## 8. 工具栏动作

- 标题栏提供核心文件与视图动作入口。
- 标题栏提供常用 Markdown 格式化入口。
- 标题栏提供 `More` 菜单承载补充动作。
- 插入标题。
- 插入粗体。
- 插入斜体。
- 插入行内代码。
- 插入代码块。
- 插入链接。
- 插入图片。
- 插入无序列表。
- 插入有序列表。
- 插入引用。
- 提供视图模式切换。
- 提供深浅主题切换。

## 9. 预览能力

- 预览区实时渲染 Markdown。
- 编辑区与预览区支持同步滚动。
- 支持标题、段落、列表、引用。
- 支持粗体、斜体、链接、图片。
- 支持行内代码和代码块。
- 支持表格。
- 预览区滚动行为稳定。

## 10. 双栏交互

- 默认左编辑右预览。
- 支持拖拽调整分栏宽度。
- 限制最小宽度，避免任一栏不可用。
- 允许切换到仅编辑或仅预览。
- 持久化视图模式与分栏比例。

## 11. 代码组织

- 编辑器面板单独成组件。
- 预览面板单独成组件。
- 工具栏单独成组件。
- 文档状态与文件 IO 分离。
- Markdown 渲染配置独立抽离。
- 文本格式化工具函数独立抽离。

## 12. 样式组织

- 优先使用 Tailwind 工具类。
- 语义色优先，不直接在业务组件里散落硬编码颜色。
- 公共变量集中管理。
- 代码字体与正文字体区分。
- 深浅主题共享同一套语义变量命名。

## 13. 基础验证

- 启动后默认看到双栏布局。
- 左侧输入时右侧预览实时更新。
- 打开本地 Markdown 文件后内容正确加载。
- 保存后 `isDirty` 状态清除。
- 修改后能正确再次标记为未保存。
- 工具栏插入动作能作用到当前文本。
- 分栏拖拽后布局正常。
- 仅编辑和仅预览模式可切换。
- 主题切换后标题栏、面板、预览和状态栏配色一致。

## 14. 文档维护

- 实现前更新 `README.md`。
- 功能变化时同步更新实现功能文档。
- 验收标准变化时同步更新 checklist。
